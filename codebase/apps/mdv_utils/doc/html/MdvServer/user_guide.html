MdvServer User's Guide

This document describes how run the MdvServer process and how to set up clients to use the server.

1. Command Line

The MdvServer is started with the following command line: 

MdvServer options

where the following options are valid:

-help
print the command usage to stdout.

-port number
the port number to be used by the server. This option MUST be specified or the MdvServer will exit.

-qmax seconds
the maximum quiescent time in seconds. The server will exit if no requests are received within this time period. If -qmax isn't specified on the command line, the server will run indefinitely.

-params file
default parameter file to be used by this server. The server will use this parameter file unless there is a different parameter file specified in the URL or there is a .MdvServer.default parameter file in the data store directory.

-print_params
Print the default parameter file to stdout.
The -port, -qmax and -params options are supported so that this server can be managed by the DIDSS (Data Ingest and Distributed Server System) server manager.

When running the MdvServer within the DIDSS, do NOT start the process by hand. Instead, let the server manager start the process. Any MdvServer process started by hand within the DIDSS will be ignored by the rest of the system and a redundant server will be started by the server manager.

2. Client Set-up

Setting a client up to use an MdvServer is as easy as using the appropriate RAP URL in the client. Generally, the URL will be specified in the client's parameter file.

To have the client go through the DIDSS data mapper and server manager to find the data, leave the host and port specifications blank in the URL. The client will then get the host name from the data mapper and then get the port number from the server manager.

To use a particular MdvServer (generally one that was started by hand), specify both the host and the port in the URL.
